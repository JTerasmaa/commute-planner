# My Commute Planner Due to CORS restrictions on both the Digitransit and Transport Tallinn APIs, we need a small server‑side proxy. The easiest way is to deploy a **Cloudflare Worker** that forwards GraphQL queries (and/or SIRI XML) with proper CORS headers. Then our static GitHub Pages site can call that worker directly. --- ## 1. Cloudflare Worker (proxy.js) Create a new Worker in your Cloudflare dashboard (or via Wrangler) with this `proxy.js`: ```js addEventListener('fetch', event => { event.respondWith(handle(event.request)); }); async function handle(req) { const url = new URL(req.url); // Proxy GraphQL queries if (url.pathname === '/graphql') { const digitransit = 'https://api.digitransit.ee/routing/v1/routers/estonia/index/graphql'; // Pass through body & headers const resp = await fetch(digitransit, { method: req.method, headers: { 'Content-Type': 'application/json' }, body: await req.text(), }); const body = await resp.text(); return new Response(body, { status: resp.status, headers: { 'Content-Type': 'application/json', 'Access-Control-Allow-Origin': '*', }, }); } // Proxy SIRI XML for trams (optional) if (url.pathname === '/siri') { const stopid = url.searchParams.get('stopid'); const source = `https://transport.tallinn.ee/siri-stop-departures.php?stopid=${stopid}`; const resp = await fetch(source); const xml = await resp.text(); return new Response(xml, { status: resp.status, headers: { 'Content-Type': 'application/xml', 'Access-Control-Allow-Origin': '*', }, }); } return new Response('Not found', { status: 404 }); } ``` - Publish the worker (e.g. at `https://commute-proxy.your-domain.workers.dev`). --- ## 2.
